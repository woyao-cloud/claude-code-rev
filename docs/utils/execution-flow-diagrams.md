### AWS 认证刷新流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        AWS 认证刷新流程                                       │
└─────────────────────────────────────────────────────────────────────────────┘

refreshAwsAuth(awsAuthRefresh)
    │
    ▼
1. 创建 AwsAuthStatusManager 实例
   └── 开始跟踪认证状态
       │
       ▼
2. 执行 awsAuthRefresh 命令
   │
   ├── stdout 事件
   │   └── 添加输出到状态管理器（UI 显示）
   │       └── logForDebugging(output)
   │
   ├── stderr 事件
   │   └── 设置错误状态
   │       └── logForDebugging(error, { level: 'error' })
   │
   └── close 事件
       │
       ├── code === 0 (成功)
       │   ├── logForDebugging('AWS auth refresh completed successfully')
       │   ├── authStatusManager.endAuthentication(true)
       │   └── resolve(true)
       │
       └── code !== 0 (失败)
           │
           ├── signal === 'SIGTERM' (超时)
           │   ├── 显示超时消息（3 分钟）
           │   └── resolve(false)
           │
           └── 其他错误
               ├── 显示错误消息
               └── resolve(false)
```

### 路径验证流程图

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            路径验证流程                                      │
└─────────────────────────────────────────────────────────────────────────────┘

validatePath(path, cwd, context, operationType)
    │
    ▼

       │
       ▼
2. 安全检查
   │
   ├── UNC 路径检查
   │   └── containsVulnerableUncPath() → 拒绝
   │
   ├── 波浪号变体检查
   │   └── cleanPath.startsWith('~') → 拒绝
   │
   ├── Shell 展开语法检查
   │   ├── 包含 '$' → 拒绝
   │   ├── 包含 '%' → 拒绝
   │   └── 以 '=' 开头 → 拒绝
   │
   └── Glob 模式检查
       ├── 写/创建操作 + Glob → 拒绝
       └── 读操作 + Glob → 验证基础目录
           │
           ▼
3. 路径解析
   │
   ├── 转换为绝对路径
   │   └── isAbsolute() ? path : resolve(cwd, path)
   │
   └── 安全解析路径
       └── safeResolvePath()
           ├── 阻止 UNC 路径
           ├── lstat 检查特殊文件类型
           ├── realpathSync 解析符号链接

               ▼
4. 权限检查
   │
   ├── isPathAllowed(resolvedPath, context, operationType)
   │   │
   │   ├── 拒绝规则匹配 → 拒绝

   │   │
   │   ├── 路径安全检查 → 拒绝（如果危险）
   │   │
   │   ├── 工作目录检查
   │   │   └── 在目录内 + acceptEdits → 允许
   │   │
   │   ├── 内部可读路径检查（读操作）→ 允许（如果匹配）
   │   │
   │   ├── 沙箱白名单检查 → 允许（如果匹配）
   │   │
   │   ├── 允许规则匹配 → 允许
   │   │
   │   └── 默认 → 拒绝

   └── 返回 { allowed, resolvedPath, decisionReason }
```

## Git 操作流程

### Git 根目录查找流程图

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        Git 根目录查找流程（带 LRU 缓存）                       │
└─────────────────────────────────────────────────────────────────────────────┘

findGitRoot(startPath)
    │
    ▼
1. 检查 LRU 缓存
   │
   ├── 命中 → 返回缓存结果
   │
   └── 未命中 → 继续
       │
       ▼
2. 初始化查找
   │
   ├── current = resolve(startPath)
   ├── root = 根目录（如 '/' 或 'C:\'）
   └── statCount = 0
       │
       ▼
3. 向上遍历目录树
   │
   ┌─────────────────────────────────────────────┐
   │ while current !== root:                     │
   │   │                                         │
   │   ▼                                         │
   │   检查 .git 是否存在                          │
   │   │                                         │
   │   ├── 存在且是目录/文件 → 找到 Git 根           │
   │   │   ├── 记录日志（duration, stat_count）    │
   │   │   ├── 缓存结果                          │
   │   │   └── 返回 current                      │
   │   │                                         │
   │   └── 不存在 → 继续向上                      │
   │       │                                     │
   │       ▼                                     │
   │   current = dirname(current)                │
   │                                             │
   │   检查是否到达根目录                          │
   │   │                                         │
   │   ├── 是 → 跳出循环                          │
   │   └── 否 → 继续循环                          │
   └─────────────────────────────────────────────┘
       │
       ▼
4. 检查根目录
   │
   ├── .git 存在 → 返回根目录
   │
   └── .git 不存在 → 返回 null
       │
       ▼
5. 缓存结果（LRU，最大 50 条目）
   │
   └── 返回结果
```

### Worktree 规范根目录解析

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Worktree 规范根目录解析流程                                │
└─────────────────────────────────────────────────────────────────────────────┘

resolveCanonicalRoot(gitRoot)
    │
    ▼
1. 读取 .git 文件内容
   │
   ├── 读取 join(gitRoot, '.git')
   │
   ├── 不是文件（是目录） → 常规仓库，返回 gitRoot
   │
   └── 是文件 → 继续
       │
       ▼
2. 检查 gitdir 前缀
   │
   ├── 不以 'gitdir:' 开头 → 返回 gitRoot
   │
   └── 以 'gitdir:' 开头 → 解析 worktree
       │
       ▼
3. 解析 worktree gitdir
   │
   ├── worktreeGitDir = resolve(gitRoot, gitdir 内容)
   │
   └── 继续
       │
       ▼
4. 读取 commondir
   │
   ├── 读取 join(worktreeGitDir, 'commondir')
   │
   ├── 读取失败 → 返回 gitRoot（子模块）
   │
   └── 读取成功 → 继续
       │
       ▼
5. 安全验证
   │
   ├── 验证 1: worktreeGitDir 是 <commonDir>/worktrees/ 的子目录
   │   │
   │   ├── 不匹配 → 返回 gitRoot（恶意仓库）
   │   │
   │   └── 匹配 → 继续
   │
   └── 验证 2: gitdir 指回 gitRoot/.git
       │
       ├── 不匹配 → 返回 gitRoot（借用攻击）
       │
       └── 匹配 → 继续
           │
           ▼
6. 处理 Bare 仓库
   │
   ├── basename(commonDir) !== '.git'
   │   │
   │   ├── 是 → 返回 commonDir（Bare 仓库工作树）
   │   │
   │   └── 否 → 返回 dirname(commonDir)
   │
   └── 返回规范根目录
```

### TTL 缓存（同步）流程

```

└─────────────────────────────────────────────────────────────────────────────┘

memoizeWithTTL(f, cacheLifetimeMs)
    │
    ▼
1. 生成缓存键
   │
   └── key = jsonStringify(args)
       │
       ▼
2. 检查缓存
   │
   ├── 未命中
   │   │
   │   ├── 计算 f(...args)
   │   ├── cache.set(key, { value, timestamp: now, refreshing: false })
   │   └── 返回 value
   │
   └── 命中 → 继续
       │
       ▼
3. 检查过期
   │
   ├── 未过期 (now - timestamp < cacheLifetimeMs)
   │   │
   │   └── 返回 cached.value
   │
   └── 已过期且未刷新 → 继续
       │
       ▼
4. 后台刷新
   │
   ├── cached.refreshing = true
   │
   ├── Promise.resolve().then()
   │   │
   │   ├── newValue = f(...args)
   │   ├── 如果缓存项仍是同一个 → 更新缓存
   │   └── 错误时删除缓存项
   │
   └── 返回 cached.value（立即）
```
