## 工具调用完整流程

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                          工具调用完整流程                                        │
└─────────────────────────────────────────────────────────────────────────────────┘

模型返回 tool_use
│
├─► 1. 解析工具调用
│   │
│   └─► {
│         id: "toolu_xxx",
│         name: "Bash",
│         input: { command: "ls -la" }
│       }
│
├─► 2. 查找工具实现
│   │
│   ├─► 内置工具？
│   │   └─► BashTool, FileReadTool, FileWriteTool, etc.
│   │
│   ├─► MCP 工具？
│   │   └─► mcp__server__tool 前缀
│   │
│   └─► 插件工具？
│       └─► 插件注册的工具
│
├─► 3. 权限检查
│   │
│   ├─► checkPermissions(toolName, args)
│   │   │
│   │   ├─► 获取规则
│   │   │   ├── getAllowRules()
│   │   │   └── getDenyRules()
│   │   │
│   │   ├─► 匹配规则
│   │   │   ├── 工具名匹配
│   │   │   └── 参数匹配
│   │   │
│   │   ├─► 分类器检查 (auto 模式)
│   │   │   └── bashClassifier / yoloClassifier
│   │   │
│   │   └─► Hook 检查
│   │       └── executePermissionRequestHooks()
│   │
│   └─► 决策
│       ├── allow → 继续执行
│       ├── deny → 拒绝，返回错误
│       └── ask → 询问用户
│
├─► 4. 执行工具
│   │
│   ├─► tool.execute(args)
│   │   │
│   │   ├─► BashTool
│   │   │   ├── spawn subprocess
│   │   │   ├── stream stdout/stderr
│   │   │   └── wait for exit
│   │   │
│   │   ├─► FileReadTool
│   │   │   ├── fs.readFile()
│   │   │   └── 返回内容
│   │   │
│   │   ├─► FileWriteTool
│   │   │   ├── fs.writeFile()
│   │   │   └── 返回成功
│   │   │
│   │   └─► 其他工具...
│   │
│   └─► 捕获输出
│       └─► ToolResult { data, newMessages }
│
├─► 5. 构建工具响应
│   │
│   └─► {
│         type: "tool_result",
│         tool_use_id: "toolu_xxx",
│         content: [
│           { type: "text", text: "输出内容" }
│         ]
│       }
│
└─► 6. 发送回 API (下一轮查询)
    │
    └─► query(messages + tool_result)
        │
        └─► 继续对话...
```

## 错误处理链路

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                          错误处理链路                                            │
└─────────────────────────────────────────────────────────────────────────────────┘

API 错误
│
├─► 1. SDK 层错误处理
│   │
│   ├─► APIConnectionError
│   │   └─► 网络错误、超时
│   │
│   ├─► APIUserAbortError
│   │   └─► 用户取消请求
│   │
│   └─► APIStatusError
│       ├── 529 Overloaded
│       ├── 429 Rate Limit
│       ├── 401 Unauthorized
│       ├── 403 Forbidden
│       └── 5xx Server Error
│
├─► 2. withRetry 层错误处理
│   │
│   ├─► 错误分类
│   │   ├── 可重试错误
│   │   └── 不可重试错误
│   │
│   ├─► 重试决策
│   │   ├── isRetryableError()
│   │   ├── shouldRetry()
│   │   └── isTransientCapacityError()
│   │
│   ├─► 特殊处理
│   │   ├── 529 连续错误追踪
│   │   ├── 快速模式降级
│   │   └── 持久重试模式
│   │
│   └─► 重试延迟
│       └─► getRetryDelay() - 指数退避 + 抖动
│
├─► 3. 业务逻辑层错误处理
│   │
│   ├─► CannotRetryError
│   │   └─► 重试耗尽，返回用户友好错误
│   │
│   ├─► FallbackTriggeredError
│   │   └─► 降级模型触发
│   │
│   └─► 工具执行错误
│       └─► 捕获并转换为 tool_result
│
└─► 4. 应用层错误处理
    │
    ├─► 显示错误消息
    │   └─► 用户友好的错误提示
    │
    ├─► 记录错误日志
    │   └─► logEvent('api_error', { ... })
    │
    └─► 清理状态
        └─► clearSession() / clearAllSessions()
```
